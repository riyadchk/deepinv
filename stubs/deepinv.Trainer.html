Trainer class for training a reconstruction network.

Note

The losses can be chosen from the libraries‚Äô training losses, or can be a custom loss function, as long as it takes as input (x, x_net, y, physics, model) and returns a scalar, where x is the ground reconstruction, x_net is the network reconstruction \(\inversef{y, A}\), y is the measurement vector, physics is the forward operator and model is the reconstruction network. Note that not all inpus need to be used by the loss, e.g., self-supervised losses will not make use of x.

Parameters:

  • model (torch.nn.Module, deepinv.models.ArtifactRemoval) ‚Äì Reconstruction network, which can be PnP, unrolled, artifact removal or any other custom reconstruction network.

  • train_dataloader (torch.utils.data.DataLoader) ‚Äì Train dataloader.

  • epochs (int) ‚Äì Number of training epochs.

  • losses (torch.nn.Module, list of torch.nn.Module) ‚Äì Loss or list of losses used for training the model.

  • eval_dataloader (torch.utils.data.DataLoader) ‚Äì Evaluation dataloader.

  • physics (deepinv.physics.Physics, list[deepinv.physics.Physics]) ‚Äì Forward operator(s) used by the reconstruction network at train time.

  • optimizer (torch.nn.optim) ‚Äì Torch optimizer for training the network.

  • grad_clip (float) ‚Äì Gradient clipping value for the optimizer. If None, no gradient clipping is performed.

  • scheduler (torch.nn.optim) ‚Äì Torch scheduler for changing the learning rate across iterations.

  • device (torch.device) ‚Äì gpu or cpu.

  • ckp_interval (int) ‚Äì The model is saved every ckp_interval epochs.

  • eval_interval (int) ‚Äì Number of epochs between each evaluation of the model on the evaluation set.

  • img_interval (int) ‚Äì Frequency of plotting images to wandb during test evaluation (at the start of each epoch)

  • save_path (str) ‚Äì Directory in which to save the trained model.

  • verbose (bool) ‚Äì Output training progress information in the console.

  • unsupervised (bool) ‚Äì Train an unsupervised network, i.e., uses only measurement vectors y for training.

  • plot_images (bool) ‚Äì Plots reconstructions every ckp_interval epochs.

  • wandb_vis (bool) ‚Äì Use Weights & Biases visualization, see https://wandb.ai/ for more details.

  • wandb_setup (dict) ‚Äì Dictionary with the setup for wandb, see https://docs.wandb.ai/quickstart for more details.

  • online_measurements (bool) ‚Äì Generate the measurements in an online manner at each iteration by calling physics(x). This results in a wider range of measurements if the physics‚Äô parameters, such as parameters of the forward operator or noise realizations, can change between each sample; these are updated with the physics.reset() method. If online_measurements=False, the measurements are loaded from the training dataset

  • plot_measurements (bool) ‚Äì Plot the measurements y. default=True.

  • check_grad (bool) ‚Äì Check the gradient norm at each iteration.

  • ckpt_pretrained (str) ‚Äì path of the pretrained checkpoint. If None, no pretrained checkpoint is loaded.

  • fact_losses (list) ‚Äì List of factors to multiply the losses. If None, all losses are multiplied by 1.

  • freq_plot (int) ‚Äì Frequency of plotting images to wandb during train evaluation (at the end of each epoch). If 1, plots at each epoch.
